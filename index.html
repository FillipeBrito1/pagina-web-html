Introdução ao HTML na Prática

(voltar)

Nesta primeira aula, "Introdução ao HTML na Prática", o foco foi em como adicionar mais controle e funcionalidades aos elementos de uma página. Aprendi sobre atributos essenciais de tags, como o id para criar um identificador único para um elemento, a class para agrupar diferentes elementos sob um mesmo estilo ou comportamento, o style para aplicar CSS diretamente em uma tag e o type para especificar o tipo de um elemento. Além disso, explorei o funcionamento dos links, entendendo como usar target="_blank" para abrir um link em uma nova aba e target="_self" para abri-lo na mesma página, garantindo uma melhor experiência de navegação para o usuário.

Trabalhando com Formulários em HTML

(voltar)

A segunda aula, "Trabalhando com Formulários em HTML", foi focada em como coletar informações do usuário de maneira estruturada. O aprendizado começou com a tag principal <form>, entendendo seus atributos cruciais como method (para definir como os dados são enviados, via GET ou POST), target para decidir onde a resposta será exibida, e autocomplete para controlar o preenchimento automático do navegador. Também vi como usar eventos, como o onsubmit, para acionar ações no momento do envio.

Explorei a versatilidade da tag <input>, descobrindo tipos modernos que vão além do texto simples, como type="color" para um seletor de cores, type="range" para uma barra de seleção, type="date" para um calendário e type="file" para permitir o upload de arquivos. Aprendi a diferença fundamental entre checkbox, que permite múltiplas seleções, e os radio buttons, para escolher uma única opção dentro de um grupo. Para listas de opções, utilizei a tag <select> em conjunto com as tags <option>, criando menus dropdown. Finalmente, para campos de texto de múltiplas linhas, como caixas de comentários, aprendi a usar a tag <textarea>.

Estruturando Seu HTML + Formatações

(voltar)

Na terceira aula, "Estruturando seu HTML + Formatações", aprofundei meus conhecimentos em como organizar o conteúdo e aplicar ênfase visual. Na parte de formatação de textos, aprendi a utilizar tags como <b> para deixar o texto em negrito, <i> para itálico, <u> para sublinhado e a própria tag <mark> para criar um destaque, como um marca-texto.

Para a estrutura da página, entendi o papel das tags <div>, um container genérico em bloco usado para agrupar seções maiores de conteúdo, e <span>, um container em linha para manipular pequenos trechos de texto sem quebrar a linha. Também vi como a tag <fieldset> é útil para agrupar campos relacionados dentro de um formulário, melhorando a organização. Por fim, aprendi a usar a tag <iframe>, que possibilita incorporar uma outra página da web diretamente dentro do meu documento atual.

Trabalhando com Mídias utilizando HTML

(voltar)

Na quarta aula, "Trabalhando com Mídias utilizando HTML", o foco foi em como enriquecer a página com elementos multimídia. Comecei pela tag <img>, aprendendo a inserir imagens e a importância de seus atributos: o title, que exibe um texto de ajuda ao passar o mouse por cima, e principalmente o alt, que fornece um texto alternativo essencial para a acessibilidade e para quando a imagem não pode ser carregada.

Em seguida, explorei as tags de áudio e vídeo. Com a tag <audio>, entendi como exibir os controles de reprodução com o atributo controls e como iniciar o áudio automaticamente com autoplay. Vi também que é uma boa prática usar a tag filha <source> para oferecer diferentes formatos do mesmo áudio, garantindo compatibilidade com mais navegadores. Aprendi que a tag <video> funciona de maneira muito similar para incorporar vídeos. Para tornar as mídias ainda mais acessíveis, utilizei a tag <track>, que serve para adicionar legendas, e seu atributo srclang para especificar o idioma do arquivo de legenda.

Criando Tabelas com HTML

(voltar)

Na quinta aula, "Criando Tabelas com HTML", aprendi a estruturar dados em formato de linhas e colunas. A base de tudo é a tag <table>, que serve como o contêiner principal. Dentro dela, cada linha é definida pela tag <tr> (table row). As células são criadas com <td> (table data) para dados comuns e com <th> (table header) para as células de cabeçalho, que dão título às colunas.

Para uma organização mais semântica e robusta, vi como agrupar as seções da tabela com as tags <thead> para o cabeçalho, <tbody> para o corpo principal de dados, e <tfoot> para o rodapé. Para dar um título oficial à tabela, utilizei a tag <caption>. Também explorei atributos da tag <table> para estilização, como border, width, align="center", cellspacing (espaço entre células) e cellpadding (espaço dentro das células). Por fim, aprendi sobre o atributo summary, importante para a acessibilidade, pois descreve o conteúdo da tabela para leitores de tela.

Entendendo HTML Semântico

(voltar)

Na sexta aula, "Entendendo HTML Semântico", o foco foi em usar tags que dão significado e estrutura ao conteúdo, o que é fundamental para acessibilidade e para tecnologias como o web scraping. Entendi como um crawler (robô de busca) navega pela página e como uma estrutura semântica clara facilita essa leitura e melhora o ranqueamento nos buscadores.

Aprendi a montar o esqueleto de uma página moderna usando as tags de layout: <header> para o cabeçalho, <nav> para a área de navegação principal, <main> para abrigar o conteúdo central e único, <aside> para conteúdos laterais (como uma sidebar), e <footer> para o rodapé. Para agrupar conteúdos relacionados dentro do corpo principal, vi a utilidade da tag <section> para criar seções temáticas e da <article> para conteúdos independentes e completos, como um post de blog.

Para conteúdos mais específicos, aprendi a diferenciar a tag <blockquote>, para formatar um bloco de citação longa, da tag <q>, para citações curtas inseridas no meio de um parágrafo. Por fim, explorei como associar mídias, como imagens e diagramas, a uma legenda, usando o conjunto das tags <figure> (para a mídia) e <figcaption> (para a legenda). Também vi o poder da tag <picture>, que permite carregar diferentes imagens de acordo com o tamanho da tela, otimizando a experiência do usuário.
